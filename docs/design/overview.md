# 概要: 背景・目的・スコープ

## 1. 背景と課題

### 現状

GitHub Actionsのワークフローファイルで使用するアクションのバージョン管理は手動で行っている。新しいバージョンがリリースされても気づきにくく、セキュリティパッチの適用が遅れるリスクがある。

### 課題・ペイン

- 課題1: ワークフローファイル内のアクションが最新かどうか、ファイルを見ただけでは分からない
- 課題2: コミットSHAで固定されたアクションが、どのタグに対応するか追跡が困難
- 課題3: バージョンの更新作業が手動でGitHubリポジトリのリリースページを確認する必要がある

### 解決しない場合の影響

セキュリティパッチの適用漏れ、古いバージョンの使用継続によるCI/CDの非効率化。

---

## 2. 目的とゴール

### 目的

VS Codeエディタ上でGitHub Actionsのアクションバージョン情報をインライン表示し、バージョン切り替えを可能にする拡張機能を開発する。

### 成功指標

| 指標                     | 現状値 | 目標値 | 計測方法                 |
| ------------------------ | ------ | ------ | ------------------------ |
| バージョン確認の所要時間 | 数分   | 即座   | ファイル表示時に自動表示 |
| 古いバージョンの検出率   | 0%     | 100%   | 装飾表示によるカバー率   |

### スコープ外

- Docker参照（`docker://...`）のバージョン管理
- ローカルアクション（`./...`）の解析
- GitHub Marketplace APIとの連携（将来対応予定）
- Dependabot連携（将来対応予定）

---

## 3. スコープ

### 対象ユーザー

- プライマリ: GitHub Actionsワークフローを管理する開発者
- セカンダリ: CI/CDパイプラインの保守担当者

### 機能一覧

| 優先度 | 機能                         | 説明                                             |
| ------ | ---------------------------- | ------------------------------------------------ |
| Must   | バージョン情報インライン表示 | uses:行にバージョンステータスを行末装飾で表示    |
| Must   | バージョン切り替え           | QuickPickによるバージョン選択とテキスト自動置換  |
| Must   | SHA⇔タグ相互変換             | コミットハッシュとタグ名の相互変換               |
| Must   | キャッシュ機構               | GitHub APIレスポンスのメモリ+永続キャッシュ      |
| Should | ホバー詳細表示               | ホバー時にバージョン詳細とコマンドリンクを表示   |
| Should | GitHub認証                   | PAT/gh CLI/環境変数によるAPI認証                 |
| Could  | CodeLens表示                 | 行の上にバージョン変更リンクを表示（オプション） |
| Won't  | 一括アップデート             | ワークスペース全体の一括更新（今回は対応しない） |
